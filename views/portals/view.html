Criterios de todo el portal

  • Uso de identidad oficial {%if evaluations[0].ease_portal_navigation_criteria.oficial_identity %} Si {%else%} No {%endif%}

    Para aumentar la confiabilidad de la información publicada, la entidad responsable del portal debe ser fácil de identificar, es necesario utilizar logo o isotipo de la entidad (municipio, ministerio, institución) en todas las páginas.

  • Enlace al sitio oficial {%if evaluations[0].ease_portal_navigation_criteria.link_oficial_site %} Si {%else%} No {%endif%}

    Para facilitar la verificación de la relación del portal con la entidad, el sitio web de la entidad responsable del portal debe ser fácil de acceder. Es necesario colocar un enlace al sitio web de la entidad (municipio, ministerio, institución) en todas las páginas.

  • Explicación de datos abiertos {%if evaluations[0].ease_portal_navigation_criteria.open_data_exp %} Si {%else%} No {%endif%}

    Para muchos ciudadanos, el concepto de datos abiertos es novedoso, y por eso es necesario incluir una descripción de este concepto en la portada, o alternativamente en una página dedicada. El texto debe tener más de 20 palabras, mencionar algunos de los beneficios de los datos abiertos.

  • Enlace a todos los datasets {%if evaluations[0].ease_portal_navigation_criteria.all_dataset_link %} Si {%else%} No {%endif%}

    Para facilitar la comprensión de todo el contenido del portal, el listado completo de todos los conjuntos de datos debe poder ser accedido, no un listado parcial.

  • Sistema de búsqueda {%if evaluations[0].ease_portal_navigation_criteria.dataset_search_system %} Si {%else%} No {%endif%}

    Para facilitar la búsqueda de conjuntos de datos y recursos en el portal, debe existir un sistema de búsqueda que permita utlizar nombres, etiquetas y contenido para encontrar conjuntos de datos y devolver resultados relevantes.

  • Existencia de API {%if evaluations[0].automated_portal_use_criteria.existence_api %} Si {%else%} No {%endif%}

    Para facilitar el acceso automatizado a los conjuntos de datos se requiere una interfaz especial para que los sistemas accedan a los datos. El portal debe tener una API REST. La API puede requerir el uso de un Token, siempre y cuando no sea necesario registrarse para obtenerlo.

  • Documentación de la API {%if evaluations[0].automated_portal_use_criteria.api_documentation %} Si {%else%} No {%endif%}

    Para que los desarrolladores de software puedan fácilmente desarrollar sistemas que consuman datos del portal, es necesario un documento que detalle el funcionamiento de la API REST.
